Spacing

Choose spacing based on how closely related elements are:

  • .m-1, .p-1 – Component internals (icon + text, tight groupings)
  • .m-2, .p-2 – Elements within a component and most paddings. (form inputs, list items, card content)
  • .m-4, .p-4, g-4 – Between components, gaps (cards, layout rhythm)
  • .m-5, .p-5 – Page-level structure and breathing room
  • Typically horizontal spacing is larger than vertical to create a more open, readable layout. gx-3 gy-2 or px-3 py-2 are common patterns.

Boxed content

Use borders, backgrounds, and shadows to establish visual hierarchy and component boundaries:

  • .border – Use borders to define component boundaries in UI-driven layouts (cards, forms, sidebars). Content-focused pages rely on whitespace and typography instead.
  • .rounded-2 – Apply to interactive containers (clickable cards, buttons) and floating elements (modals, dropdowns) for a modern, approachable feel.
  • .rounded-pill – Use for prominent call-to-action elements that need maximum visual emphasis.
  • .shadow – Apply to floating elements (modals, dropdowns) or interactive cards on hover to establish elevation hierarchy.
