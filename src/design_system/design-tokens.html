Spacing

Choose spacing based on how closely related elements are:

  • .m-1, .p-1 – Component internals (icon + text, tight groupings)
  • .m-2, .p-2 – Elements within a component and most paddings. (form inputs, list items, card content)
  • .m-4, .p-4, g-4 – Between components, gaps (cards, layout rhythm)
  • .m-5, .p-5 – Page-level structure and breathing room
  • Typically horizontal spacing is larger than vertical to create a more open, readable layout. gx-3 gy-2 or px-3 py-2 are common patterns.

Borders, Corners & Elevation

Use borders and shadows to establish visual hierarchy and component boundaries:

  • .border – Subtle separation between sections or to define component edges
  • .rounded-2 – Default corner rounding for most components (cards, buttons, inputs)
  • .shadow-sm – Subtle elevation for interactive elements on hover
  • .shadow – Standard elevation for cards and raised components
  • .shadow-lg – Strong elevation for modals, dropdowns, and floating elements
