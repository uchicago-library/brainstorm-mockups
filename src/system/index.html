Design System Documentation

The University of Chicago Library Design System includes design principles, visual guidelines, components, and patterns.

Overview

This design system is based on the official University of Chicago Library brand guidelines and is designed to be reusable across multiple platforms and services. It balances consistency with flexibility to support diverse use cases.

Design Atomization Levels

This design system is organized into five levels of abstraction:

A. Design Tokens

Colors, typography, spacing, shadows, borders, transitions

B. Components

Buttons, forms, cards, accordions, nav, tables, modals, badges

C. Patterns

Component combinations: hero sections, mini-layouts, listings

D. Layouts & Templates

Full page templates with grid, responsive behavior, page types

E. Documentation

Guidelines, usage examples, contribution guides, validation

Vision: Multi-Platform Support

This design system is architected to support distribution across multiple platforms with varying CSS footprints:

  • Main output: Full design system with all components
  • Platform-specific outputs: LibGuides, Catalog, Book a Room, etc.
  • Minimal output: Lightweight builds for platforms needing only core styles

See Methodology for technical implementation details.

Use Cases

This design system aims to accommodate the following use cases. Implementation strategies are documented in Methodology.

  • Repository - Digital collections platform
  • LibGuides - Research and subject guides
  • Book a Room - Facility reservation system
  • Catalog - Library catalog interface
  • Other library services and websites
