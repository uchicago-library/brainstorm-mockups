Project Methodology

Goals

  • Consistency and modularity across platforms (Wagtail, VuFind, SpringShare, Aeon, FindIt!, etc)
    • Identical elements across platforms like tabs, search buttons, thumbnails, …
    • Maybe have a set of labels for subjects
  • We have a small team, so it can't be too demanding to maintain.
  • Easy for new or temporary contributors (like fellows, new staff, GenAI) to maintain consistency. For example, easily answer when someone should create a new class, use a Bootstrap class or find one that is available and make changes to it.
  • Distinguish editor documentation and developer documentation.
  • Use atomic design for scalability and sustainability. We want to keep building on this design system instead of doing one redesign every 10-15 years.
  • We will be using the latest Bootstrap and Font Awesome with SCSS to customize variables.
