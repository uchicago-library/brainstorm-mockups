Adding a New Component

Step-by-step guide to creating reusable components with proper structure and documentation.

Component Overview

A component is a reusable UI element (button variant, card style, form input, etc.) that:

  • Has dedicated SCSS styling
  • Is documented in a demo page
  • Follows naming conventions and BEM methodology
  • Can be copied into other projects

Step 1: Create the SCSS File

Create a new file in src/styles/components/:

src/styles/components/_component-name.scss

Naming: Use kebab-case with underscore prefix (e.g., _search-box.scss)

Template Example:

// Search Box Component
.search-box {
  display: flex;
  position: relative;
  border: 1px solid $border-subtle;
  border-radius: $border-radius;
  padding: $spacing-xs;
}

.search-box__input {
  flex: 1;
  border: none;
  outline: none;
  font-size: 1rem;
  padding: $spacing-sm;
  
  &::placeholder {
    color: $text-muted;
  }
}

.search-box__button {
  padding: $spacing-sm $spacing-md;
  background: $primary;
  color: white;
  border: none;
  border-radius: $border-radius;
  cursor: pointer;
  
  &:hover {
    background: darken($primary, 10%);
  }
}

// Modifier for expanded state
.search-box—expanded {
  .search-box__suggestions {
    display: block;
  }
}

Step 2: Import the File

Add your component to src/styles/main.scss in the components section:

// Components
@import "components/component-name";

Make sure the import appears after base styles and variables.

Step 3: Create Demo HTML

Create a demo page in src/design_system/ or src/design_system/components/:

src/design_system/components/demo-component-name.html

Step 4: Update Navigation

Add a link to your new demo in relevant index pages:

  • src/design_system/components/index.html (if you created a component demo)
  • src/index.html (if it's a major component)

Example Link:

<div class="card">
  <div class="card-body">
    <h3 class="h5 card-title">
      <a href="/brainstorm-mockups/design_system/components/demo-search-box/">Search Box</a>
    </h3>
    <p class="card-text">A reusable search input component.</p>
  </div>
</div>

Step 5: Verify & Test

  1. Run dev server: npm run dev
  2. Navigate to your demo page and verify styles are applied correctly
  3. Test responsive behavior: Check mobile, tablet, and desktop views
  4. Check accessibility: Use axe DevTools to audit for issues
  5. Validate HTML/CSS: Use W3C Validator

Best Practices

  • Use Bootstrap utilities where appropriate instead of custom CSS
  • Follow BEM naming for components (see Naming Conventions)
  • Use SCSS variables for all colors, spacing, etc.
  • Make it reusable: Avoid one-off styles or hard-coded values
  • Test on multiple browsers: Chrome, Firefox, Safari
  • Document thoroughly: Explain the purpose and usage with examples
  • Keep it simple: Avoid over-engineering; reuse Bootstrap components

For Mockup/Experimental Components

If you're experimenting with a new component in a mockup file, you can:

  1. Use a <style> block directly in the mockup HTML (not inline)
  2. Keep styles centralized at the top of the file
  3. Once finalized, move styles to a dedicated SCSS file
