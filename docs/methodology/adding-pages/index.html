Adding a New Page

Create design system demos and mockup pages using reusable components.

Types of Pages

Page Type Location Purpose Example
Component Demo src/design_system/ Show a single component in all its variations buttons.html, forms.html
Page Mockup src/design_mockups/ Full-page layouts combining components guides-in-accordions.html
Documentation src/methodology/ or src/design_system/ Guided documentation for development Setup, best practices, principles

Best Practices

  • Use semantic HTML: <header>, <main>, <section>, <article>
  • Keep pages modular: Use partial includes for repeated elements
  • Test responsiveness: Use Bootstrap's grid and utilities
  • Include code examples: Show developers how to use components
  • Document variations: Show different states and options
  • Add descriptions: Explain the purpose of each section
  • Maintain hierarchy: Use proper heading levels (h1, h2, h3...)

Working with Experimental/Mockup Components

For mockup pages with experimental designs:

  • Use a <style> block at the top of the HTML file
  • Keep styles centralized and commented
  • Once the design is finalized, move styles to a dedicated SCSS file
  • Update the component organization
