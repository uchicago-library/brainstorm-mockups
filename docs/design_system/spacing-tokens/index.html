Spacing Tokens

Principles for choosing spacing utilities that create consistent, harmonious layouts at scale.

Spacing Decision Framework

Choose spacing based on how closely related the elements are:

Component Interior (`.m-1`, `.p-1`)

Tightest spacing for elements that form a single UI unit.

Tightly grouped elements

When to use: Icon + text in a button, items inside a list, form field internals, badge + count

Element to Element (`.m-2`, `.p-2`) ‚Äî Most Common

Default spacing between elements within a component or section.

Element 1
Element 2

When to use: Between inputs in a form, list items, button groups, cards in a grid, content inside a card body

Component to Component (`.m-4`, `.p-4`) ‚Äî Most Common

Standard spacing between distinct components or major content blocks.

Component Block 1
Component Block 2

When to use: Between separate cards, section padding, gap between major content areas, page structure rhythm

Section & Page Level (`.m-5`, `.p-5`)

Generous spacing for top-level page structure and breathing room.

Page section with breathing room

When to use: Page padding, gap between major sections, hero to content transition, footer spacing

Extreme Cases (`.m-3`, `.m-6`, etc.)

Use when the standard levels don't fit. Intentional deviations for special situations.

When to use: Subtle adjustments between `.m-2` and `.m-4`, or extra breathing room beyond `.m-5`

Quick Reference Table

Spacing Class Use Case Examples Frequency
.m-1, .p-1 Component internals Icon + text, tight groupings Occasional
.m-2, .p-2 Element to element Form inputs, list items, card content Very Common
.m-4, .p-4 Component to component Card spacing, section padding, layout rhythm Very Common
.m-5, .p-5 Page-level breathing Page padding, major section gaps Common
.m-3, .m-6, etc. Fine-tuning Adjustments when standard levels don't fit Rare

Decision Flow

1. Ask: Are these elements part of the same component?
  • Yes, tightly grouped: Use .m-1 or .p-1
  • Yes, loosely grouped: Use .m-2 or .p-2
  • No, different components: ‚Üí Go to step 2
2. Ask: Are these components close together (same row/section)?
  • Yes: Use .m-4 or .p-4
  • No, far apart (major sections): Use .m-5 or .p-5
3. If nothing feels right: Use .m-3 or .m-6 for fine-tuning

Margin vs Padding

Use Margin (.m-*)

For spacing outside a component:

  • Spacing between components
  • Pushing elements away from siblings
  • Section gaps and layout rhythm

Use Padding (.p-*)

For spacing inside a component:

  • Content breathing room inside a box
  • Card bodies and containers
  • Page/section containers
